# Grid-Connected PV System with Boost Converter and Three-Phase Inverter

A MATLAB/Simulink model of a three-phase, grid-connected photovoltaic (PV) power system. The model simulates the full power path from solar irradiance to grid injection: a DC-DC boost stage with Maximum Power Point Tracking (MPPT), a regulated DC bus, and a grid-tied three-phase voltage source inverter (VSI) with dq0 current control.

## Overview

The system is built around three control layers working together:

1. **MPPT (Perturb & Observe)** — extracts maximum available power from the PV array by adjusting the boost converter's duty cycle in real time.
2. **DC Bus Voltage Control** — regulates the DC link voltage to a fixed reference (800 V) via a PI controller, generating the active current reference for the grid-side inverter.
3. **Grid-Side Current Control (dq0)** — controls active and reactive power injected into the grid by regulating the d-axis and q-axis current components, using a PLL for grid synchronization.

## System Architecture

```
PV Array → Boost Converter → DC Bus → Three-Phase VSI → LC/LCL Filter → Grid
   │              │                         │
   │              │                         └── dq0 current control (id, iq)
   │              └── MPPT (P&O) → duty cycle
   └── Irradiance & temperature inputs
```

### 1. PV Array & Boost Stage
- PV array block driven by an irradiance profile (Signal Editor) and a fixed cell temperature.
- Boost converter (IGBT switch, inductor, output capacitor) steps up the PV array voltage to the DC bus level.
- Duty cycle is generated by the MPPT controller and applied via a PWM generator (`PWM_Boost`).

### 2. MPPT — Perturb & Observe
- Implemented as a MATLAB Function block (`mppt_po`).
- Inputs: PV voltage (`Vpv`), PV current (`Ipv`), and adjustable perturbation step size (`deltaD_in`).
- Output: duty cycle command for the boost converter.
- Continuously perturbs the operating point and observes the resulting change in power to converge on the PV array's maximum power point.

### 3. DC Bus Voltage Regulation
- DC bus voltage (`Vbus`) is measured and compared to a fixed reference (`Vbus_ref = 800 V`).
- A discrete PI controller (`PI(z)`) processes the error and outputs the **d-axis current reference** (`id_ref`) for the grid-side controller — this ties the DC bus energy balance directly to how much active power the inverter pushes into the grid.
- A supervisory comparator (`Vbus ≥ 1.05 × Vbus_ref`) provides an overvoltage protection/limiting condition.

### 4. Grid-Side dq0 Current Control
- Grid voltages and currents (`Vabc`, `Iabc`) are transformed into the rotating dq0 reference frame, synchronized via a PLL that extracts grid frequency/angle (`wt`).
- Two independent PI current controllers regulate:
  - **d-axis current** (`id_ref` vs. measured `i_d`) → active power
  - **q-axis current** (`q_ref = 0` vs. measured `i_q`) → reactive power (set for unity power factor operation)
- Controller outputs (`d_m`, `q_m`) are transformed back to the abc frame to generate the inverter's modulation reference (`m`).

### 5. Three-Phase Inverter & Grid Interface
- The modulation index drives a PWM generator producing switching pulses (`pulse_inv`) for the three-phase VSI.
- Inverter output passes through an LC filter before connecting to the grid through a transformer.
- Active and reactive power delivered to the grid (`Pinv`, `Qinv`) are computed from measured `Vabc`/`Iabc`.

### 6. Monitoring & Power Quality
- Total Harmonic Distortion (THD) of the grid current is computed and scaled to a percentage output (`THDI`).
- PV-side instantaneous power (`Ppv`) is calculated from `Vpv × Ipv`.

## Model Details

| Parameter | Value |
|---|---|
| Solver | Discrete, fixed-step, 1e-5 s |
| DC bus voltage reference | 800 V |
| Reactive current reference | 0 A (unity power factor) |
| MPPT step size (deltaD) | 125e-6 |
| PV temperature | Fixed at 25 °C |

## Requirements

- MATLAB / Simulink (developed and tested on R2024b or later — uses the Signal Editor block)
- Simscape Electrical (Specialized Power Systems) for the PV array, power switches, and three-phase measurement blocks

## How to Run

1. Open `<Grid-connected PV with boost converter and inverter>.slx` in Simulink.
2. Set or edit the irradiance profile in the Signal Editor block to the desired test scenario.
3. Run the simulation.
4. View results on the scope(s): PV voltage/current, DC bus voltage, injected grid current, active/reactive power, and THD.

## Possible Future Improvements

- Variable/dynamic PV cell temperature input instead of a fixed value.
- Anti-islanding protection logic.
- LVRT (Low Voltage Ride-Through) behavior under grid fault conditions.
- Adaptive MPPT step size for faster tracking with less steady-state oscillation.

## Notes

This project was built and debugged iteratively in Simulink, including resolving PV array numerical stability at small discrete time steps (Robust Discrete Model), PWM modulation-index scaling for grid synchronization, and closed-loop dq0 current control tuning.
